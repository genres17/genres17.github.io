docker容器概述

容器：容器是一种轻量级，可移植并将应用程序打包的技术，使得程序可以在任何地方以相同的方式运行。

  • docker将镜像运行起来的就是容器，容器相当于动态的镜像，类似exe文件和运行程序的关系
  • 容器具备一定生命周期
  • 可以使用docker ps命令查看容器
docker容器与虚拟机相同点
  • 容器和虚拟机一样，都需要裸机的物理资源分配（内存，cpu)
  • 容器和虚拟机的生命周期比较相似，都需要经过创建、运行、暂停、关闭
  • 容器和虚拟机都能安装各种应用，如redis等
  • 容器和虚拟机都会在主机操作系统下创建文件夹保存信息
docker容器与虚拟机
  • 容器是直接运行在 宿主机上的，本质上是宿主机一系列进程集合，但是虚拟机是在虚拟化技术基础上运行完整操作系统的
  • docker是轻量级的，虚拟机是重量级的，虚拟机的花费要更大，docker则很小。
虚拟机的生命周期

  • running和paused状态下虚拟机还在运行下，对于虚拟机文件删除只能采用强制删除方式，其他情况则可以采用正常删除手段

docker容器生命周期管理

容器创建

作用：利用镜像创建一个待使用的容器

命令格式：docker create [OPTIONS] IMAGE [COMMAND] [ARG…]

常用参数（这个参数太多了）：

-t 分配一个虚拟终端

-i 使用stdin打开

–name 为创建的容器起个别名

注：[COMMAND]和[ARG..]指的是可选的image里的命令和可选参数，如image是Ubuntu，command可以使用ls -al，这样生成的容器会默认执行该命令（初始化执行），如果不指定该命令，则会执行默认的，要查看默认命令，通过docker inspect命令查看cmd项的值。参数-t,-i一般一并使用，因为打开的镜像内需要使用终端操作，不加会导致打开失败

例如：

docker create -ti centos python3

容器删除

作用：删除一个或者多个容器

命令格式：docker rm [OPTIONS] 容器1，容器2

常用参数：

-f 强制删除容器

例如：

docker rm -f centos_python

容器启动

作用：从创建状态或者退出状态启动一个容器

命令格式：docker start [OPTIONS] 容器1，容器2

常用参数：

-a 将容器启动的内容标准输出到当前控制台

-i 把当前标准输入传递给容器

–rm 在容器停止后就立即删除容器

例如:

docker create -a centos_ls

**注：上图所示的是启动centos并执行ls -als的情况，如果启动的时候不加-a参数，则不会将容器的输出输出到当前终端上，若是还想对容器里的终端进行操作，则启动还必须加上-i **

容器运行

作用：创建并启动容器

命令格式：docker run [OPTION] 镜像 [COMMAND] [ARG]

常用参数：-ti （同create）

例如：

docker run -ti centos python

小结：
  1. docker run xxx 等价于 docker create+docker start -a
  2. docker run -d 等价于docker create +docker start(后台运行，只要加了-d一律都是后台运行了)
  3. 拿不准想要控制态输出，就写docker run -ti xxx
容器关闭

作用：关闭一个运行的容器(若不指定-t时间则默认等10s)

命令格式：docker stop [OPTION] 容器名

可选参数：

-t 后面接int数字，表示在第x秒后执行关闭操作

例如：

docker stop -t 511 8514

容器强制关闭（不推荐使用，除非真的关不上容器）

作用：强制关闭容器

命令格式：docker kill [OPTION] 容器名

可选参数：

-s 给出kill的信号，具体参考如下Linux信号

注：默认情况下，docker stop命令先执行的是第15 SIGTERM信号，处理完后再执行SIGKILL信号，docker kill直接执行的是SIGKILL信号

例如：

docker kill 9091
容器暂停/继续

作用：暂停容器运行/解除暂停状态

命令格式：docker pause xxx/docker unpause xxxx

例如：

docker pause 4050
docker unpause 4050

容器重启

作用：重启容器

命令格式：docker restart 容器名

可选参数：

-t 在第n秒后开始重启

例如：

docker restart 5112
docker容器信息查询

命令同镜像信息查询

dockers容器日志

这里所谓的日志，就是启动容器后指定的容器的cmd命令输出的内容

命令格式：docker logs 容器名

可选参数：

-f 持续输出，指那些一直有输出的日志信息

-t 带有时间的输出

例如：

docker logs -t cf92

docker容器重命名

作用：给容器重命名

例如：

docker rename xxxx test

容器连接

作用：将当前终端与标准输入输出流连接

命令格式：docker attach 容器名

例如：

docker attach 0857

注：退出容器的终端除了使用终端自带的exit外，还可以通过ctrl+d退出

容器执行新的命令

作用: 类似于在容器里新开了一个终端，不影响容器命令本体

命令格式：docker exec -ti 容器名 命令

常用参数：

-d 后台运行

例如：

docker exec -ti 1234 python
